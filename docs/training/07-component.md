# Stuart Sierra Component -- Lifecycle Management

[Component](https://github.com/stuartsierra/component) manages the lifecycle of stateful resources (database connections, message broker connections, background threads). It starts them in dependency order and stops them in reverse order.

**Dependency:** `com.stuartsierra/component {:mvn/version "1.1.0"}`

## Namespace Used

```clojure
(:require [com.stuartsierra.component :as component])
```

---

## Core Concepts

### Components

A component is a Clojure record that implements the `Lifecycle` protocol:

```clojure
(defrecord Datasource [mode jdbc-url user password image
                       ;; runtime state (set during start)
                       container datasource]
  component/Lifecycle

  (start [this]
    (case mode
      :testcontainers
      (let [pg (infra/start-postgres!)
            ds (infra/->datasource pg)]
        (assoc this :container pg :datasource ds))

      :jdbc-url
      (let [ds (jdbc/get-datasource {:jdbcUrl jdbc-url})]
        (assoc this :datasource ds))))

  (stop [this]
    (when (and (= :testcontainers mode) container)
      (infra/stop-postgres! container))
    (assoc this :container nil :datasource nil)))
```

**Key rules:**
- `start` returns a new version of `this` with runtime state added
- `stop` returns a new version of `this` with runtime state cleared
- Both methods are immutable -- they return new records, not mutate
- `start` acquires resources; `stop` releases them

### Factory Functions

Convention: provide a factory function that creates the component from config:

```clojure
(defn datasource-component
  [{:keys [mode jdbc-url user password image]}]
  (map->Datasource {:mode     mode
                    :jdbc-url jdbc-url
                    :user     user
                    :password password
                    :image    image}))
```

`map->Datasource` is auto-generated by `defrecord` -- it creates a Datasource from a map.

## System Map

A system is a collection of named components with declared dependencies:

```clojure
(defn dev-system []
  (component/system-map
    :datasource (datasource-component {:mode :testcontainers})
    :migrator   (component/using
                  (migrator-component)
                  [:datasource])))    ;; migrator depends on :datasource
```

### `component/system-map` -- Declare Components

Creates a system map. Each key is a component name, each value is a component.

```clojure
(component/system-map
  :event-store-ds  (datasource-component {:mode :testcontainers})
  :read-db-ds      (datasource-component {:mode :testcontainers})
  :rabbitmq        (rabbitmq-component {:mode :testcontainers})
  :event-migrator  (component/using (migrator-component) {:datasource :event-store-ds})
  :outbox-poller   (component/using (outbox-poller-component {...})
                     {:datasource :event-store-ds
                      :rabbitmq   :rabbitmq}))
```

### `component/using` -- Declare Dependencies

Tells Component which other components to inject before starting:

```clojure
;; Vector form: component field name must match system key
(component/using (migrator-component) [:datasource])
;; Before starting migrator, inject the :datasource component into its :datasource field

;; Map form: rename dependencies
(component/using (outbox-poller-component {...})
                 {:datasource :event-store-ds    ;; field :datasource gets :event-store-ds
                  :rabbitmq   :rabbitmq})        ;; field :rabbitmq gets :rabbitmq
```

## Start and Stop

### `component/start` -- Start the System

Starts all components in dependency order:

```clojure
(def system (component/start (dev-system)))
;; 1. Starts :event-store-ds (no dependencies)
;; 2. Starts :read-db-ds (no dependencies)
;; 3. Starts :rabbitmq (no dependencies)
;; 4. Starts :event-migrator (depends on :event-store-ds)
;; 5. Starts :outbox-poller (depends on :event-store-ds, :rabbitmq)
```

### `component/stop` -- Stop the System

Stops all components in **reverse** dependency order:

```clojure
(component/stop system)
;; 1. Stops :outbox-poller (depends on others, so stop first)
;; 2. Stops :event-migrator
;; 3. Stops :rabbitmq
;; 4. Stops :read-db-ds
;; 5. Stops :event-store-ds (nothing depends on it, so stop last)
```

## Components in This Project

### Datasource

Manages a PostgreSQL connection (Testcontainers or JDBC URL):

```clojure
(datasource-component {:mode :testcontainers})
(datasource-component {:mode :jdbc-url :jdbc-url "jdbc:postgresql://..." :user "..." :password "..."})
```

Access the datasource after start: `(:datasource started-component)`

### Migrator

Runs database migrations on start. Depends on a Datasource:

```clojure
(migrator-component)                                    ;; default migration dir
(migrator-component {:migration-dir "read-migrations"}) ;; custom dir
```

### RabbitMQConnection

Manages a RabbitMQ connection (Testcontainers, URI, or direct):

```clojure
(rabbitmq-component {:mode :testcontainers})
(rabbitmq-component {:mode :uri :uri "amqp://..."})
```

### OutboxPoller

Polls the outbox table and publishes to RabbitMQ. Depends on Datasource + RabbitMQ:

```clojure
(outbox-poller-component {:exchange-name "events" :poll-interval-ms 100})
```

### AsyncProjector

Subscribes to a RabbitMQ queue and projects events to a read database. Depends on two Datasources + RabbitMQ:

```clojure
(async-projector-component
  "projector.account-balances"
  {:projection-name   "account-balances"
   :read-model-tables ["account_balances"]
   :handler           my-handler})
```

## Full System Example

```clojure
;; bank/components.clj
(defn dev-full-system []
  (component/system-map
    ;; Databases
    :event-store-ds (datasource-component {:mode :testcontainers})
    :read-db-ds     (datasource-component {:mode :testcontainers})

    ;; Migrations
    :event-migrator (component/using (migrator-component)
                      {:datasource :event-store-ds})
    :read-migrator  (component/using (migrator-component {:migration-dir "read-migrations"})
                      {:datasource :read-db-ds})

    ;; Messaging
    :rabbitmq       (rabbitmq-component {:mode :testcontainers})

    ;; Outbox publisher
    :outbox-poller  (component/using
                      (outbox-poller-component {:exchange-name "events"})
                      {:datasource :event-store-ds :rabbitmq :rabbitmq})

    ;; Async projectors
    :account-projector
    (component/using
      (async-projector-component "projector.accounts" account-config)
      {:event-store-ds :event-store-ds
       :read-db-ds     :read-db-ds
       :rabbitmq       :rabbitmq})))
```

## Accessing Component State

After starting, components are records with populated fields:

```clojure
(let [system (component/start (dev-system))
      ds     (get-in system [:event-store-ds :datasource])]
  ;; ds is now a JDBC datasource, ready to use
  (jdbc/execute! ds ["SELECT 1"]))
```

## Dependency Injection Pattern

Component injects dependencies by `assoc`-ing them into the record before calling `start`. Inside `start`, you access them by field name:

```clojure
(defrecord Migrator [migration-dir
                     datasource]     ;; <-- injected by Component
  component/Lifecycle
  (start [this]
    (let [ds (:datasource datasource)]  ;; access the injected component
      (migrations/migrate! ds :migration-dir (or migration-dir "migrations"))
      (assoc this :migrated true)))
  (stop [this]
    (assoc this :migrated nil)))
```

Note: `datasource` here is the **Datasource component** (a record), not a JDBC datasource. You extract the JDBC datasource via `(:datasource datasource)`.

## Error Handling in Stop

Stop methods should use try-finally to ensure cleanup even if something throws:

```clojure
(stop [this]
  (try
    (when poller
      (outbox/stop-poller! poller))
    (finally
      (when channel
        (rabbitmq/close-channel channel))))
  (assoc this :channel nil :poller nil))
```

This ensures the channel is closed even if stopping the poller throws.
